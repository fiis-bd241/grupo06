### ***Plan de Próximos pasos la Optimización de la Aplicación Web***

***Proyecto: Optimización de Aplicación Web para Empresa Textil Vircatex***

***Fecha de Inicio: 15 de Julio, 2024***

***Responsable del Proyecto: Grupo 6***


## Introducción

Este documento presenta la hoja de ruta para la mejora continua de la aplicación web desarrollada con React, Django y PostgreSQL, con planes futuros para integrar ClickHouse. La aplicación está diseñada para satisfacer las necesidades operativas de una empresa textil. Tras la primera versión, el objetivo es contrastar los requerimientos iniciales y planificar mejoras a largo plazo utilizando el marco de trabajo Scrum. [![Scrum](https://img.shields.io/badge/Scrum-Agile-blue.svg)](https://www.scrum.org/)

## Objetivos

- Asegurar que la aplicación web cumple con los requerimientos iniciales y las expectativas del cliente.
- Planificar y ejecutar mejoras continuas en la funcionalidad y rendimiento de la aplicación.
- Implementar un módulo de seguridad y un modelo de autenticación.
- Preparar la infraestructura para la integración futura de ClickHouse junto a PostgreSQL.
- Mantener una comunicación constante y efectiva con el cliente para la retroalimentación y ajustes necesarios.
- Implementar el marco de trabajo Scrum para gestionar el desarrollo y las mejoras.
- Incluir testers para revisar y asegurar la calidad de la aplicación.

## Metodología Scrum

El proyecto se gestionará utilizando Scrum, una metodología ágil centrada en la entrega incremental y la adaptación continua. Se trabajará en iteraciones llamadas Sprints, cada una con una duración de dos semanas. El equipo de desarrollo participará en eventos clave de Scrum: Sprint Planning, Daily Standups, Sprint Review y Sprint Retrospective.

### Roles Scrum

- **Product Owner (PO):** Responsable de maximizar el valor del producto y gestionar el backlog del producto.
- **Scrum Master (SM):** Facilita el proceso Scrum y ayuda al equipo a trabajar de manera efectiva.
- **Development Team:** Grupo multifuncional encargado de entregar incrementos del producto al final de cada sprint.
- **Testers:** Dos testers dedicados a revisar y asegurar la calidad de la aplicación mediante pruebas exhaustivas.

| Tarea                                 | Descripción                                                                                                             |
|---------------------------------------|-------------------------------------------------------------------------------------------------------------------------|

## Características para Optimización Web

### Optimización de Código

| Característica            | Descripción                                                                                                           |
|---------------------------|-----------------------------------------------------------------------------------------------------------------------|
| **Lazy Loading**          | - Carga diferida de imágenes y componentes no críticos.                                                               |
| **Memoization**           | - Uso de memoization para optimizar el rendimiento de componentes React.                                              |
| **Código Split**          | - División del código en fragmentos para cargar solo lo necesario.                                                    |
| **Minificación y Compresión** | - Minificación de archivos CSS y JS.<br>- Compresión de recursos estáticos utilizando Gzip.                         |
| **Optimización de Consultas** | - Indexación de bases de datos.<br>- Uso eficiente de consultas SQL y ajuste de consultas complejas.                |

## Pruebas y Aseguramiento de la Calidad

### Rol de los Testers

| Tarea                        | Descripción                                                                                                             |
|------------------------------|-------------------------------------------------------------------------------------------------------------------------|
| **Planificación de Pruebas** | - Definir casos de prueba y escenarios de uso.                                                                          |
